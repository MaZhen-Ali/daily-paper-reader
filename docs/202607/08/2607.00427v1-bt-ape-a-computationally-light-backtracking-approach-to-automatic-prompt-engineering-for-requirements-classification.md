## 摘要
大语言模型 (LLMs) 越来越多地应用于需求工程 (RE) 任务，但引导它们的提示词通常是通过试错法手动设计的，导致结果不一致且并非最优。自动化提示词构建在需求工程领域仍处于探索阶段，其有效性尚不明确。为了解决这一问题，我们提出了一种轻量级的自动提示工程方法——回溯 APE (BT-APE)，并将其应用于需求分类。我们将提示词设计建模为一个优化问题，通过 LLM 生成的候选方案、回溯搜索和动态示例选择来迭代优化提示词。我们在三个基准数据集上使用五种经过指令微调的 LLM 对 BT-APE 进行了评估，并将其与四种经典的提示词基准（零样本、少样本、思维链、思维链+少样本）以及一种最先进但资源密集型的 APE 基准 (PE2) 进行了比较。BT-APE 和 PE2 达到了几乎相同的准确率，且均以显著的效应量大幅优于经典基准；然而，BT-APE 的计算开销要小得多，在同等准确率下，其输入的 token 减少了约 72%，实际运行时间减少了 66%，因此更适合资源受限的部署环境。我们的贡献主要有三点：(i) 一个轻量级的 APE 框架，并附带开放的交互式工具和复现包；(ii) 首次在需求分类任务中对 APE 与经典提示词方法进行了系统性比较；(iii) 深入探讨了类别定义和提示词演化对性能的影响。

## Abstract
Large language models (LLMs) are increasingly applied to requirements engineering (RE) tasks, yet the prompts guiding them are typically designed manually through trial and error, yielding inconsistent and suboptimal results. Automated prompt construction remains largely unexplored in RE, leaving its effectiveness unclear. To address this, we propose a lightweight Automatic Prompt Engineering approach, Backtracking APE (BT-APE), and apply it to requirements classification. We frame prompt design as an optimization problem, iteratively refining prompts via LLM-generated candidates, backtracking search, and dynamic example selection. Evaluating BT-APE on three benchmark datasets with five instruction-tuned LLMs, we compare it against four classical prompting baselines (zero-shot, few-shot, chain-of-thought, CoT+few-shot) and a state-of-the-art but resource-intensive APE baseline (PE2). BT-APE and PE2 achieve nearly identical accuracy, both substantially outperforming the classical baselines with large effect sizes; however, BT-APE imposes a far lighter computational footprint, consuming roughly 72% fewer input tokens and 66% less wall-clock time at equivalent accuracy, making it better suited to resource-constrained deployment. Our contributions are threefold: (i) a lightweight APE framework with an open interactive tool and replication package; (ii) the first systematic comparison of APE against classical prompting for requirements classification; and (iii) insights into how class definitions and prompt evolution affect performance.

在需求工程（RE）中，利用大语言模型（LLM）进行需求分类（如区分功能性与非功能性需求）已成为趋势。然而，现有的提示词（Prompt）设计大多依赖于人工试错，缺乏系统性优化，导致模型性能不稳定且难以复现。虽然通用自然语言处理（NLP）领域已出现自动提示工程（APE）技术，但在特定领域的需求工程任务中，这些方法往往计算开销巨大（如 PE2 方法），且其在需求分类任务上的有效性尚未得到系统验证。本研究旨在开发一种**计算轻量级**的自动提示优化方法，以在保证分类准确率的同时，降低对计算资源（Token 消耗和时间）的需求。

### 2. 方法论：BT-APE 框架
BT-APE 的核心思想是将提示词设计视为一个**受限的搜索优化问题**，通过迭代反馈来精炼提示词。
*   **核心流程**：
    1.  **初始化**：将数据集分为示例池（30%）、验证集（30%）和测试集（40%）。
    2.  **迭代精炼**：LLM 根据当前提示词和一组平衡的示例（包含分类正确和错误的案例）生成新的候选提示词。
    3.  **回溯搜索（Backtracking）**：引入“耐心值”（Patience Threshold, $X=3$）。如果连续 $X$ 次迭代未能提升验证集上的 F1 分数，算法将回溯到历史排名次优的提示词重新开始搜索，防止陷入局部最优或无效路径。
    4.  **动态示例选择**：每轮迭代从示例池中抽取“1个正确正例、1个正确负例、1个错误正例、1个错误负例”，确保模型能同时接收到成功和失败的信号。
    5.  **多数投票机制**：在推理时采用 3 次运行的多数投票，以抵消 LLM 的随机性。

### 3. 实验设计
*   **数据集**：使用了三个经典的需求分类基准：**PROMISE (NFR)**、**PROMISE (Refined)** 和 **SecReq**（安全需求分类）。
*   **对比方法（Baselines）**：
    *   **经典提示策略**：Zero-shot（零样本）、Few-shot（少样本）、CoT（思维链）、CoT + Few-shot。
    *   **SOTA 自动优化方法**：**PE2**（一种基于历史条件的 APE 方法，作为主要的性能与效率对比基准）。
*   **实验模型**：选择了 5 个 7B-8B 参数规模的开源指令微调模型：Qwen2-7B、Falcon3-7B、Granite-3.2-8B、Ministral-8B 和 LLaMA-3-8B。

### 4. 资源与算力
*   **硬件环境**：实验在单台配备 **NVIDIA H200 (142 GB)** GPU 的服务器上运行。

    *   **单次延迟**：BT-APE 的提示词生成延迟比 PE2 降低了约 60%。

### 5. 实验数量与充分性
*   **实验规模**：总共进行了 **15 组核心配置**（3 个数据集 × 5 个模型），每组配置均运行了 5 个不同的随机种子以确保稳定性。
*   **充分性评价**：
    *   **统计严谨性**：采用了 Wilcoxon 符号秩检验、Friedman 检验及线性混合效应模型（LMM）进行多维度分析，并进行了 Holm–Bonferroni 校正。
    *   **消融与敏感性分析**：对回溯阈值 $X$ 和迭代次数 $N_{max}$ 进行了敏感性分析，并对比了“领域知识初始化”与“无信息初始化”的影响。
    *   **公平性**：所有方法在相同的硬件、相同的测试集划分和相同的投票协议下进行评估，实验设计非常客观、公平。

### 6. 主要结论与发现
1.  **性能卓越**：BT-APE 在所有数据集和模型上均显著优于四种经典提示词基准（效应量大且具有统计学意义）。
2.  **效率极高**：在分类准确率（F1 分数）与 PE2 持平的情况下，BT-APE 的计算资源消耗大幅降低。
3.  **任务难度主导**：数据集的复杂程度（如四分类任务 vs 二分类任务）对性能的影响远大于模型选择。
4.  **提示词特征**：高效的提示词通常具有**简洁、动作导向（动词多）、结构清晰（标点符号多）**的特点，而过高的词汇多样性和句法复杂性反而有害。

*   **轻量化设计**：通过回溯机制和受限上下文，解决了 APE 方法在长序列迭代中 Token 爆炸的问题。

*   **工具化产出**：论文发布了一个基于 GUI 的交互式工具，降低了非专家使用自动提示工程的门槛。
