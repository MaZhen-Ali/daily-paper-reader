## 摘要
面向对象的需求编写与管理方法 (OOMRAM) 是一种需求复用框架，它依赖于精确的标识符匹配和僵化的模板，限制了其在不同语境下调整规范的能力。虽然大语言模型 (LLM) 提供了克服这一瓶颈的灵活性，但它们也带来了生成结构无效或不一致的需求组合的风险。为了解决这一矛盾，我们提出了一种神经符号多智能体系统，将需求复用重新概念化为模型驱动的启发过程。在这种范式中，LLM 作为一种非确定性启发式方法，用于遍历由形式化 OOMRAM 需求格表示的确定性领域模型。一个确定性的符号验证器在智能体循环中强制执行所有结构约束，从而通过构造有效地消除了幻觉需求组合。在跨两个应用系列的自主基准测试中，我们的系统实现了 100% 的需求覆盖率，且约束违反率仅为 0.2%。尽管针对单一金标准的 F1 分数适中 (0.47-0.51)，但生成的每个规范在结构上都是有效的，并满足所有强制性的领域需求。这种与模型无关的实现通过子图导航扩展到更大的格，并为监管合规性提供透明的审计追踪。

## Abstract
The Object-Oriented Method for Requirements Authoring and Management (OOMRAM) is a requirements reuse framework that relies on exact identifier matching and rigid templates, limiting its ability to adapt specifications across diverse contexts. While Large Language Models (LLMs) offer the flexibility to overcome this bottleneck, they introduce the risk of generating structurally invalid or inconsistent requirement combinations. To address this tension, we present a neuro-symbolic multi-agent system that re-conceptualizes requirements reuse as a Model-Driven Elicitation process. In this paradigm, an LLM serves as a non-deterministic heuristic for traversing a deterministic domain model represented by a formal OOMRAM requirement lattice. A deterministic, symbolic validator enforces all structural constraints within the agent loop, effectively eliminating hallucinated requirement combinations by construction. Evaluated on an autonomous benchmark across two application families, our system achieves 100% requirement coverage and a constraint-violation rate of only 0.2%. Although the F1-score against a single gold standard is moderate (0.47-0.51), every generated specification is structurally valid and satisfies all mandatory domain requirements. The model-agnostic implementation scales to larger lattices via subgraph navigation and provides transparent audit trails for regulatory compliance.

*   **研究动机**：需求工程（RE）中，复用已验证的需求是提高效率的关键。传统的“面向对象需求编写与管理方法”（OOMRAM）虽然严谨，但依赖精确的标识符匹配，导致用户无法使用自然语言进行灵活检索。
*   **核心矛盾**：大语言模型（LLM）虽然具备理解自然语言愿景的灵活性，但极易产生“幻觉”，例如发明不存在的需求 ID、违反强制性约束或组合互斥的需求选项。
*   **研究目标**：开发一种既能利用 LLM 的灵活性，又能通过形式化方法保证需求组合逻辑正确性的自动化系统。

### 2. 方法论：神经符号多智能体系统
*   **核心思想**：将需求复用重新定义为“模型驱动的引导（Model-Driven Elicitation）”。LLM 被视为在确定性状态空间（需求格）中进行搜索的非确定性启发式工具，而符号逻辑负责强制执行约束。
*   **关键技术细节**：
    *   **需求格（Lattice）表示**：将需求家族建模为有向无环图（DAG），定义核心（Core）、单适配器（SA，即 XOR）、多适配器（MA，即 OR）和可选（Op）四种节点类型。
    *   **多智能体架构（基于 LangGraph）**：
        *   **Navigator（导航员）**：负责在格结构中进行深度优先遍历。
        *   **Interpreter（解释员）**：由 LLM 驱动，将用户愿景映射为具体的需求 ID。
        *   **Validator（验证员）**：**纯符号算法（Python 编写，无 LLM）**。它根据形式化谓词逻辑检查 Interpreter 的输出，确保满足基数约束和父子关系。
        *   **Scribe（记录员）**：编译最终的规范文档。
    *   **子图导航策略**：每次仅让 LLM 处理当前节点的直接子节点，将上下文复杂度控制在 $O(1)$，从而支持大规模工业级需求格的线性扩展。

### 6. 主要结论与发现
*   **消除幻觉**：系统实现了 **100% 的结构有效性**。虽然 Interpreter 可能会提出错误建议，但 Validator 确保了最终输出 0 违规。
*   **约束违反率**：在运行过程中，仅出现 0.2% 的原始约束违反，且全部被 Validator 拦截并引导 Interpreter 修正。

### 7. 优点
*   **架构严谨**：通过“神经+符号”的解耦，解决了 LLM 在严肃工程领域（如安全关键系统）中不可信的问题。
*   **可扩展性强**：子图导航避免了长文本导致的 LLM 性能下降。
*   **透明可审计**：LangGraph 的状态记录提供了完整的决策追踪，符合行业监管要求。

*   **评估指标偏差**：使用单一金标准计算 F1 分数可能低估了系统的实际表现，因为需求复用本身具有多样性。
*   **依赖先验知识**：系统高度依赖于预先构建好的形式化 OOMRAM 格模型，如果底层模型有误，系统无法自纠。
*   **实验覆盖面**：目前仅使用了 8B 规模的小模型，尚未探讨更大规模模型（如 GPT-4）或更复杂工业场景下的延迟表现。
