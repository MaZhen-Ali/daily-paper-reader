## 摘要
本研究提出了一种代理式、意图驱动的端到端 (E2E) 编排框架，该框架将意图共创与测试驱动的质量保证范式相结合。在该框架中，自主智能体通过迭代将用户的初始意图细化为经过确认且可审计的规范。此外，系统在部署前自动从这些意图中导出验证测试，直接借鉴了软件工程中的测试驱动开发工作流，以确保主动的服务水平协议 (SLA) 合规性。该架构基于符合标准的知识表示，采用了 TM 论坛 (TMF) 信息模型和目录。这实现了从高级产品方案到细粒度服务/资源及测试规范的确定性图遍历。我们通过使用消息驱动的多智能体模式扩展 OpenSlice，并集成支持 MCP（模型上下文协议）的工具访问以实现实时知识检索，从而构建了该架构的原型。目前，我们对智能体的评估侧重于意图共创阶段，作为迈向全规模编排的基准。基于将多个开源大语言模型 (LLM) 后端与基于 TMF 的知识库集成的实验，我们观察到工具调用的可靠性和幻觉模式存在显著差异，这凸显了在代理式 6G 系统中进行稳健知识集成的关键重要性。

## Abstract
This work proposes an agentic, intent-driven end-to-end (E2E) orchestration framework that integrates intent co-creation with a Test-Driven Quality Assurance paradigm. In this framework, autonomous agents iteratively refine a user's initial intent into a confirmed, auditable specification. Furthermore, the system automatically derives validation tests from these intents before provisioning, directly mirroring the Test-Driven Development workflow in software engineering to ensure proactive Service Level Agreement (SLA) compliance. The architecture is grounded in a standards-aligned knowledge representation using TM Forum (TMF) information models and catalogs. This enables deterministic graph traversal from high-level Product Offerings down to granular Service/Resource and Test specifications. We prototyped this architecture by extending OpenSlice with a message-driven, multi-agent pattern and integrating MCP-enabled (Model Context Protocol) tool access for real-time knowledge retrieval. Currently, our evaluation of the agents targets the intent co-creation phase as a baseline toward full-scale orchestration. Building on experiments with multiple open-source Large Language Model (LLM) backends integrated with the TMF-based knowledge base, we observe substantial variability in tool-use reliability and hallucination patterns, underscoring the critical importance of robust knowledge integration in agentic 6G systems.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：6G 网络具有极高的异构性和动态性，传统的网络编排和质量保证（QA）面临三大挑战：
    1.  **意图模糊性**：用户意图通常是不完整或模糊的，难以直接映射到技术工作流。
    2.  **语义鸿沟**：人类语言描述与机器可执行动作之间存在断层。
    3.  **反应式 QA**：传统的“先部署后验证”模式导致 SLA 违规检测滞后。
*   **研究背景**：随着网络向高度自治（L4-L5 级）演进，需要一种能够主动验证、自动细化意图并确保合规性的智能框架。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：借鉴软件工程中的**测试驱动开发（TDD）**理念，提出**测试驱动质量保证（TDQA）**。在服务部署前，先从细化的意图自动生成验证测试，确保部署过程“赢得”合规性。
*   **关键技术细节**：
    *   **意图共创代理（Intent Co-creation Agent）**：通过 LLM 与用户迭代对话，将模糊需求转化为包含成本、质量约束的显式规范。
    *   **标准驱动的知识表示（KR）**：基于 TM Forum (TMF) 信息模型构建知识图谱，实现从“产品方案”到“服务/资源规范”再到“测试规范”的**确定性图遍历**。
    *   **双路径代理架构**：一个代理负责将意图映射为技术订单（Intent-to-Actions），另一个代理同步生成测试标准（Quality & SLO/SLA Specs）。
    *   **模型上下文协议（MCP）**：作为标准化的工具访问层，允许代理安全地调用平台功能（如查询目录、配置切片），实现“认知层”与“执行层”的分离。

### 3. 实验设计：场景、基准与对比
*   **实验场景**：以“网络即服务（NaaS）”为基准，设定一个复杂的 6G 业务需求：在希腊帕特雷市为期一周的高清体育媒体体验，要求延迟 <15ms、支持 1000 名用户、预算上限 9000 欧元。
*   **Benchmark（基准）**：建立了一个由专家定义的“地面真值（Ground Truth）”配置，包含 4 个特定产品组合（网络切片、边缘缓存、VPN、切片观测）。
*   **对比方法**：对比了 **13 种开源大语言模型**，分为“推理中心型”（如 Qwen3、Deepseek-r1）和“工具增强通用型”（如 Llama3.1、Mistral）。评估维度包括产品组合准确性、幻觉率、成本/时长计算准确性及对话耗时。

### 4. 资源与算力
*   **硬件配置**：使用了一个包含 3 个节点的 Kubernetes 集群。
    *   **CPU/RAM**：每个节点 16 vCPUs，64 GB RAM。
    *   **GPU**：其中一个节点配备了 **1 块 NVIDIA RTX A6000 (48 GB VRAM)**。
*   **推理优化**：提到可以使用 AirLLM 等技术优化边缘部署的内存需求。

### 5. 实验数量与充分性
*   **实验规模**：针对 13 个模型进行了多轮迭代对话实验（Q1-Q5 五个阶段）。
*   **充分性评价**：
    *   **客观性**：实验涵盖了不同参数规模（1.7b 到 32b）和不同架构的模型，具有较好的代表性。
    *   **局限性**：目前实验主要集中在“意图共创”这一前端阶段，尚未对后端全规模编排和实时闭环修复进行大规模压力测试。

### 6. 主要结论与发现
*   **模型表现差异巨大**：**Gpt-oss-20b** 表现最佳，能够准确估算成本、处理日期且无幻觉。
*   **工具调用是瓶颈**：许多模型（如 Deepseek-r1-32b, Llama3.1-8b）虽然推理能力强，但在通过 MCP 调用外部工具时失败或产生幻觉。
*   **幻觉风险**：小型模型（如 Smollm2, Granite）极易虚构不存在的产品 ID。
*   **知识库的重要性**：在 6G 代理系统中，稳健的知识集成（而非单纯的 LLM 推理）是确保确定性的关键。

### 7. 优点（亮点）
*   **跨界融合**：首次将软件工程的 TDD 理念引入 6G 网络编排，实现了 QA 的左移（Proactive QA）。
*   **标准化集成**：严格遵循 TM Forum 和 ETSI 标准，通过 OpenSlice 开源框架实现，具有很强的工业参考价值。
*   **架构解耦**：通过 MCP 和 Agent Skills 实现了认知逻辑与基础设施操作的解耦，增加了系统的安全性和可扩展性。

### 8. 不足与局限
*   **工具可靠性风险**：实验显示代理在调用工具时仍存在不稳定性，这在电信级应用中是致命的。
*   **实验覆盖面**：目前的评估主要基于单一的 NaaS 复杂场景，缺乏更多样化（如跨域、极端故障恢复）的场景验证。
*   **实时性挑战**：部分模型的对话和推理时间较长（约 5 分钟），可能无法满足 6G 毫秒级动态调整的需求。
