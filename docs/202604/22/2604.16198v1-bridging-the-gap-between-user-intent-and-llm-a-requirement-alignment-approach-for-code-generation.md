## 摘要
代码生成是指根据用户需求自动生成可执行程序。最近，研究人员探索了利用先进的大语言模型（LLM）来提高生成代码正确性的方法。尽管取得了一些进展，但现有方法主要集中在设计推理策略或后期优化方法，以提升代码生成性能。尽管这些方法各不相同，但它们都有一个共同的假设：LLM 能够正确理解给定的需求。然而，这一假设并不总是成立。为了填补这一空白，我们提出了 REA-Coder，这是一种旨在增强 LLM 代码生成性能的需求对齐方法。REA-Coder 首先识别与 LLM 不一致的需求内容并进行需求对齐。然后，基于对齐后的需求，LLM 生成代码并进一步验证生成的代码是否符合需求，迭代这一需求对齐和代码生成的过程，直到生成正确的代码或达到最大迭代次数。实验结果表明，REA-Coder 在五个编程基准测试中，在四种 LLM 上的表现均优于所有先进的基准模型。具体而言，REA-Coder 在五个基准数据集上分别实现了 7.93%、30.25%、26.75%、8.59% 和 8.64% 的平均提升，证明了需求对齐在提高 LLM 代码生成性能方面的有效性。

## Abstract
Code generation refers to automatically producing executable programs from user requirements. Recently, researchers have explored approaches to enhance the correctness of generated code with advanced large language models. Although achieving improvements, existing approaches focus on designing reasoning strategies or post-refinement methods to enhance code generation performance. Despite their differences, all these methods share a common assumption: the LLM can correctly understand the given requirement. However, this assumption does not always hold. To fill this gap, we propose REA-Coder, a requirement alignment approach to enhance the code generation performance of LLMs. REA-Coder involves first identifying the requirement content that does not align with LLMs and aligning the requirements. Then, based on the aligned requirements, LLMs generate code and further verify whether the generated code aligns with the requirements, iterating this process of requirement alignment and code generation until generating correct code or achieving the maximum number of iterations. Experimental results show that REA-Coder outperforms all advanced baselines on four LLMs across five programming benchmarks. Concretely, REA-Coder achieves average improvements of 7.93%, 30.25%, 26.75%, 8.59%, and 8.64% on the five benchmark datasets, demonstrating the effectiveness of requirement alignment for improving the code generation performance of LLMs.

论文的核心问题在于指出当前大语言模型（LLM）在代码生成任务中的一个**根本性盲点**：现有的优化方法（如思维链推理 SCoT、基于执行反馈的自我修复 Self-Repair）都默认 LLM 已经完全理解了用户需求。然而，作者通过分析发现，LLM 往往在需求理解阶段就产生了偏差。如果初始理解是错误的，后续的推理和修复都只是在错误的基础上“打补丁”，无法触达正确逻辑。因此，研究动机是**将需求对齐（Requirement Alignment）引入代码生成流程**，确保模型在写代码前真正“懂”了需求。

### 2. 方法论：REA-Coder 框架
REA-Coder 提出了一种迭代式的需求对齐与生成流程，包含三个核心阶段：
*   **需求对齐（Requirement Alignment）：**
    *   **维度定义：** 基于软件工程标准（如 IEEE 830），定义了需求目的、输入输出要求、边界情况、示例解释等核心维度。
    *   **QA 检查机制：** 让 LLM 根据需求生成一份“问题清单”及参考答案，随后让模型回答这些问题。通过对比模型回答与参考答案的差异，识别出模型理解偏差的点。
    *   **需求增强：** 将识别出的偏差转化为补充信息，生成一份“对齐后的增强需求”。
*   **代码生成（Code Generation）：**
    *   基于增强后的需求生成代码，并使用公共测试用例进行初步验证。
*   **需求对齐验证（Requirement Alignment Verification）：**
    *   **掩码恢复机制：** 借鉴掩码语言模型思想，将原始需求中的关键语义片段（如约束条件、算法逻辑）遮蔽（Mask），要求 LLM 根据其生成的代码来恢复这些片段。
    *   **逻辑一致性检查：** 如果恢复的内容与原需求不符，说明代码逻辑与需求脱节。这些不一致信号会反馈回第一阶段，开启新一轮迭代。

*   **性能领先：** REA-Coder 在所有设置下均达到 SOTA，在 CodeContests 等高难度竞赛题上提升尤为显著（最高达 30.25%）。

### 7. 优点（亮点）
*   **视角独特：** 突破了“模型已理解需求”的固有假设，从需求工程的角度解决代码生成问题。
*   **验证机制新颖：** 引入“掩码恢复”来验证代码与需求的一致性，这比单纯依赖测试用例更能发现深层逻辑错误。
*   **通用性强：** 作为一个 Prompting 框架，它可以即插即用地应用于任何 LLM，无需微调。

*   **成本较高：** 由于涉及多次 QA、掩码恢复和迭代，Token 消耗和推理延迟显著高于单次生成方法，不适合对实时性要求极高的场景。

*   **Prompt 敏感性：** 需求维度的提取和 QA 过程高度依赖 Prompt 设计，对于不同规模的模型，Prompt 的泛化能力可能存在波动。
